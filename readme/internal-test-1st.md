# 1차 내부 테스트 개선/수정 사항

> 테스트 일자: 2026-04-16  
> 테스트 환경: iPhone (iOS) 실기기  
> 참고: 안드로이드에서는 다른 결과일 수 있음

---

## 0. 공통사항

| 구분 | 내용 | 수정 대상 | 상태 |
|------|------|-----------|------|
| 개선 | iOS에서는 좌→우 밀어서 뒤로가기, 안드로이드에서는 back버튼에 대한 정의 — 어느 화면에서 허용하고 말지를 정해야 함 | kend-native | 미정 |
| 개선 | iOS에서 좌→우 밀어서 뒤로가기 시, 캐시 때문인지 이전 화면이 보였다가 다시 현 화면으로 돌아와 progress 이후 이전 화면으로 가는 오류 | kend-native | 미정 |
| 개선 | 스크롤이 없는 화면에서 화면이 좌우로는 밀리지 않는데, 위아래로는 밀리는 현상 (웹뷰 bounce) | kend-native | 미정 |
| 개선 | navigation 메뉴 기준 최초 1depth 페이지 말고 하위 페이지들에서 home으로 접근하기 위한 방법 필요 (상단에 홈버튼 추가) | kend | 진행중 |
| 개선 | 입력필드 선택(키패드 올라온 뒤) 시 줌이 되는데, 닫히면 다시 원래대로 돌아오도록 (줌이 안되도록) | kend + kend-native | 완료 |

## 1. 로그인 & 회원가입

| 구분 | 내용 | 수정 대상 | 상태 |
|------|------|-----------|------|
| 개선 | 로그인 페이지의 이용약관, 개인정보처리방침 버튼(링크) 화면 연결 | kend | 확인필요 |
| 개선 | 아이디찾기, 비밀번호찾기 — 휴대폰 인증이 없는 상황에서 supabase email로 인증 및 찾기하는 process 논의 필요 | kend | 논의필요 |
| 오류 | 이메일 가입하기 화면에서 하단 height가 짤려 가입하기 버튼이 잘리는 현상 | kend | 미착수 |
| 신규 | 소셜 회원가입 시 추가정보를 받아야 하는 flow 화면 추가 (닉네임, 휴대전화, 자녀정보 등) — 이메일 회원가입에도 반영 | kend | 미착수 |
